## 三、API 接口测试

### 3.1 认证

#### POST /api/admin/login

**请求体**：
```json
{ "username": "admin", "password": "Admin@123456" }
```

**成功响应**：
```json
{
  "code": 200,
  "message": "登录成功",
  "data": {
    "token": "eyJhbG...",
    "user": { "id": 1, "username": "admin", "nickname": "管理员" }

}
```

**失败用例**：
| 场景 | 请求体 | 预期结果 |
|------|--------|----------|
| 空密码 | `{ "username": "admin", "password": "" }` | `400 请输入用户名和密码` |
| 错误密码 | `{ "username": "admin", "password": "wrong" }` | `401 密码错误` |

### 3.5 订单

#### POST /api/orders — 创建订单

**门票订单**请求体：
```json
{
  "userId": 1,
  "orderType": 1,
  "itemId": 1,
  "itemName": "凤凰古城 - 成人票",
  "ticketTypeId": 1,
  "quantity": 2,
  "totalPrice": 200,
  "visitDate": "2026-07-01",
  "validDays": 1
}
```

**路线订单**请求体：
```json
{
  "userId": 1,
  "orderType": 2,
  "itemId": 1,
  "itemName": "凤凰古城两日游",
  "quantity": 2,
  "totalPrice": 600,
  "visitDate": "2026-07-01",
  "validDays": 2
}
```

**参数校验用例**：

| 场景 | 请求体 | 预期结果 |
|------|--------|----------|
| 缺 userId | `{ orderType: 1, ... }` | `400 缺少必要参数` |
| 缺 quantity | `{ userId: 1, ... }` | `400 缺少必要参数` |
| 数量为 0 | `{ ..., quantity: 0 }` | `400 数量必须大于0` |
| 负价格 | `{ ..., totalPrice: -100 }` | `400 总金额不能为负数` |
| 库存不足 | 数量 > 库存 | `400 库存不足` |
| 票种不存在 | `itemId: 9999` | `400 票种不存在` |
| 路线不存在 | `orderType: 2, itemId: 9999` | `400 路线不存在` |
| 重复下单 | 同一 userId+itemId+visitDate 已有待使用订单 | `400 您已有该日期的未使用订单` |

#### GET /api/orders/my?userId=1&page=1&pageSize=10

分页返回当前用户的订单列表。

#### 失败用例

| 场景 | 预期 |
|------|------|
| `GET /api/orders/my` (缺 userId) | `400 缺少 userId 参数` |

#### PUT /api/orders/:id/cancel

取消待使用订单。

#### GET /api/orders/:id

获取订单详情。`uuid` 字段通过 `plainify()` 已剥离，不会返回给客户端。

### 3.6 管理员接口（需 JWT）

所有 `/api/admin/*` 接口需要在请求头携带 `Authorization: Bearer <token>`。

#### 场景：未携带 Token

```bash
curl -X GET http://localhost:3000/api/admin/scenic-spots
# 预期：401 "未授权，请先登录"
```

#### 场景：Token 无效

```bash
curl -H "Authorization: Bearer invalid-token" http://localhost:3000/api/admin/scenic-spots
# 预期：401 "Token 无效或已过期"
```

#### 景区 CRUD

| 方法 | 端点 | 说明 |
|------|------|------|
| POST | `/api/admin/scenic-spots` | 创建景区。请求体包含 `name`、`coverImage`、`address`、`description`、`lat`、`lng`、`sort` |
| PUT | `/api/admin/scenic-spots/:id` | 更新景区 |
| DELETE | `/api/admin/scenic-spots/:id` | 软删除 |
| PUT | `/api/admin/scenic-spots/:id/status` | 切换状态 |

#### 票种 CRUD

| 方法 | 端点 | 说明 |
|------|------|------|
| GET | `/api/admin/ticket-types` | 全部票种（含 scenic 关系） |
| POST | `/api/admin/ticket-types` | 创建票种。请求体包含 `scenicId`、`name`、`price`、`sellPrice`、`stock`、`validDays`、`description` |
| PUT | `/api/admin/ticket-types/:id` | 更新票种 |
| DELETE | `/api/admin/ticket-types/:id` | 软删除 |
| PUT | `/api/admin/ticket-types/:id/daily-stock` | 批量设置日期库存。`{ stocks: [{ date: "2026-07-01", stock: 100 }] }` |

#### 路线 CRUD

| 方法 | 端点 | 说明 |
|------|------|------|
| GET | `/api/admin/routes` | 全部路线（含 scenic 关系） |
| POST | `/api/admin/routes` | 创建路线。请求体包含 `scenicId`、`name`、`price`、`durationDays`、`stock`、`description` |
| PUT | `/api/admin/routes/:id` | 更新路线 |
| DELETE | `/api/admin/routes/:id` | 软删除 |
| PUT | `/api/admin/routes/:id/status` | 切换状态 |
| PUT | `/api/admin/routes/:id/daily-stock` | 批量设置日期库存 |

#### 行程管理

| 方法 | 端点 | 说明 |

| POST | `/api/admin/routes/:routeId/itineraries` | 添加行程。`{ dayOrder, title, content }` |

## 七、核心业务流程测试

### 7.1 门票购买全流程

```
首页 → 景区门票 → 点击景区 → 点击票种"立即购买"
→ 弹窗选择日期 + 数量 → 确认下单
→ 跳转到"我的订单" → 查看新订单
→ 点击"取消订单" → 库存恢复
```

| 步骤 | 预期 |
|------|------|
| 库存检查 | 下单前后端校验库存，不足则拒绝 |
| 事务保护 | 库存扣减 + 订单创建在同一事务中 |
| 重复下单 | 同一用户同日期同票种不能重复下单（有待使用订单时） |
| 库存恢复 | 取消订单后，对应日期的库存恢复 |
| validDays | 从票种实体读取，非前端传入值 |

### 7.2 路线套餐购买全流程

```
首页 → 路线套餐 → 点击路线 → 选择日期 + 人数
→ 确认预订 → 跳转到"我的订单"
```

| 步骤 | 预期 |
|------|------|
| 价格计算 | 总价 = 路线价格 × 人数 |
| validDays | = 路线 durationDays |
| 行程展示 | 路线详情页展示所有行程安排 |

### 7.3 管理员后台全流程

```
登录 → 仪表盘（查看统计）
→ 景区管理（增/删/改/状态切换）
→ 票种管理（新增票种 → 设置日期库存）
→ 路线套餐（新建路线 → 添加行程 → 设置库存）
→ 订单管理（筛选 → 核销 → 退款）
```

## 八、边界 & 异常测试

### 8.1 参数校验

| 场景 | 预期 |
|------|------|
| 订单数量为 0 或负数 | `400 数量必须大于0` |
| 订单总价为负 | `400 总金额不能为负数` |
| 缺少 userId | `400 缺少必要参数` |
| 票种 / 路线不存在 | `400 票种/路线不存在` |
| 库存不足 | `400 库存不足` |

### 8.2 状态流转校验

| 操作 | 允许状态 | 拒绝状态 |
|------|----------|----------|
| 取消订单 | 0(待使用)、2(已过期) | 1(已核销)、4(已退款) |
| 退款 | 0(待使用) | 其他所有状态 |
| 核销 | 0(待使用) | 1(已核销，提示"订单已核销")、其他 |

### 8.3 并发 & 安全

| 场景 | 预期 |
|------|------|
| 并发下单 | 使用 `SELECT FOR UPDATE` 行锁，不会超卖 |
| 未登录访问 admin 接口 | `401 未授权` |
| 无效 Token 访问 admin 接口 | `401 Token 无效或已过期` |
| 订单 UUID 泄露 | `plainify()` 已剥离，客户端拿不到 UUID |
| 管理员内部错误泄露 | 已统一为 `{ code, message, data }` 格式 |

### 8.4 软删除

| 场景 | 预期 |
|------|------|
| 删除景区后查询 | `deletedAt != null`，不出现在列表中 |
| 删除景区后通过 ID 直接查 | 返回 null（`getById` 过滤 `deletedAt`） |
| 删除景区后切换状态 | `toggleStatus` 检查 `deletedAt === null`，提示"景区不存在" |
| 订单号生成 | `generateOrderNo` 排除软删除记录，不会重复或跳跃 |

| # | 限制 | 影响 |
|---|------|------|
| 1 | 无用户注册/登录系统 | 用户 ID 硬编码为 1，任何请求都可伪造 userId |
