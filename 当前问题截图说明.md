# 当前问题截图说明

## 当前可观察现象

微信开发者工具中可以打开小程序页面，页面标题显示：

```text
商品列表
```

但是页面主体区域为空白，没有显示商品列表内容。

## 已确认不是优先怀疑点

后端服务可以启动，浏览器访问接口正常：

```text
http://127.0.0.1:3000/api/app/wudong/product-categories/list
http://127.0.0.1:3000/api/app/wudong/products/page?page=1&pageSize=5
```

接口能返回 `code=1000`，商品分页接口能返回 `data.total=8` 和商品列表。

## 已尝试但未解决

已尝试在：

```text
miniapp/pages/index/index.wxml
```

加入静态调试文字或调试面板，但微信开发者工具模拟器中仍不显示。

## 截图应表达的重点

如果接手同学需要补截图，建议截图包含：

1. 微信开发者工具模拟器顶部标题为“商品列表”。
2. 页面主体空白。
3. 编辑器中当前打开的是 `miniapp/pages/index/index.wxml`。
4. 如有条件，同时截 Network 面板，确认是否发出了接口请求。

## 当前排查判断

如果 WXML 静态文字也不能显示，优先不要继续查接口，应先检查：

- 微信开发者工具导入目录是否为 `miniapp`
- `app.json` 的启动页
- `project.config.json` / `srcMiniprogramRoot`
- 当前模拟器显示的页面是否就是正在修改的 `pages/index/index`
- `pages/index/index.json` 的标题来源
- `pages/index/index.wxss` 是否隐藏或遮挡内容
