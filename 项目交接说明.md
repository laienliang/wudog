# 人工智能实践项目交接说明

## 1. 项目结构说明

项目根目录：

```bat
D:\临时文件\水课任务\人工智能实践
```

主要目录说明：

- `backend/`：后端服务，基于 Cool Admin Midway / Midway.js，提供商品分类、商品列表、商品详情、收藏等接口。
- `miniapp/`：微信小程序端，当前用于展示商品列表、商品详情、收藏等页面。
- `admin-web/`：React + Vite + Ant Design 管理后台，用于商品分类、商品、SKU、图片等运营管理页面。
- `pc-web/`：React + Vite PC Web 用户端，用于浏览商品、详情和收藏流程。
- `docs/`：课程设计、接口、联调、验收等说明文档。
- `postman/`：Postman 接口集合和本地环境配置。
- `backend/sql/`：后端初始化 SQL 和商品种子数据。

## 2. 环境要求

- Node.js：建议使用 Node.js 18 或以上版本；后端 `package.json` 声明 `node >=18.0.0`，推荐 Node.js 20 LTS。
- MySQL：建议 MySQL 8.0，当前本地配置使用 MySQL 端口 `3306`。
- 微信开发者工具：使用当前稳定版或较新版微信开发者工具，基础库可使用项目配置中的 `3.16.1` 或工具推荐版本。
- 后端端口：`3000`。
- 后端 API 前缀：`/api`。
- 数据库配置位置：
  - `backend/src/config/config.local.ts`
  - 当前本地库名：`cool`
  - 当前本地账号：`root`
  - 当前本地密码：`123456`
  - 当前本地主机与端口：`127.0.0.1:3306`

## 6. 当前未解决问题

当前未解决的问题是：

```text
后端接口有数据，但小程序页面主体空白。
```

已知现象：

- 微信开发者工具中 `miniapp` 页面能打开。
- 页面标题显示“商品列表”。
- 页面主体内容为空白。
- 已尝试在 `pages/index/index.wxml` 加静态调试文字和调试面板，但模拟器仍然不显示。

因此当前疑似问题不是后端、数据库或接口，而更可能是：

- 小程序页面入口问题
- `app.json` 启动页问题
- `project.config.json` / `miniprogramRoot` 问题
- 微信开发者工具编译路径问题
- WXML/WXSS 渲染隐藏问题
- 当前模拟器显示的页面不是正在修改的 `pages/index/index`

接手同学应优先检查：

- `app.json`
- `project.config.json`
- 启动页
- `miniprogramRoot`
- `pages/index/index.json`
- `pages/index/index.wxss`
- 是否打开了错误目录

## 7. 接手同学优先排查清单

建议按以下顺序排查：

1. `app.json` 的 `pages` 第一项是不是 `pages/index/index`。
2. 全局搜索“商品列表”，确认标题来自哪个页面。
3. `project.config.json` 的 `miniprogramRoot` 或 `srcMiniprogramRoot` 是不是正确。
4. 微信开发者工具是否导入的是 `miniapp` 目录，而不是项目根目录或其他目录。
5. `pages/index/index.wxml` 静态文字是否能显示。
6. `pages/index/index.wxss` 是否有隐藏、遮挡、透明、定位覆盖等问题。
7. 微信开发者工具 Network 面板是否有请求。
8. 如果 WXML 静态文字都不显示，先别查接口，先查页面入口。
