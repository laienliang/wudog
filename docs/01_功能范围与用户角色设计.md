# 01 功能范围与用户角色设计

本文件适用于第 1 组“衣·非遗商品模块”。

## 1. 角色定义

### 1.1 游客

* 可访问公开商品列表和商品详情
* 可浏览分类、商品图片、SKU 信息
* 不允许执行收藏、后台管理和写操作
* 不需要登录

### 1.2 已登录普通用户

* 继承游客的全部浏览能力
* 可收藏商品、取消收藏、查看我的收藏
* 需要持有普通用户 JWT
* 不允许访问管理后台接口

### 1.3 管理员

* 可登录管理后台
* 可维护分类、商品、SKU、图片
* 可执行商品上架、下架、软删除
* 需要持有管理员 JWT
* 不允许使用普通用户收藏接口代替后台接口

### 1.4 Cool 后端基础能力复用说明

* 管理员登录、JWT、角色权限、菜单和基础 CRUD 优先复用 Cool Admin Midway。
* 第 1 组不重复设计独立的管理员、角色、权限和菜单基础系统。

## 2. 功能模块划分

| 模块 | 目标 | 主要使用方 |
| --- | --- | --- |
| 商品分类 | 展示分类并支撑筛选 | 游客、普通用户、管理员 |
| 商品浏览 | 展示商品列表、搜索、筛选、分页 | 游客、普通用户 |
| SKU 选择 | 查看规格、价格、库存、售罄状态 | 游客、普通用户 |
| 商品详情 | 展示商品详情、工艺介绍、传承人信息 | 游客、普通用户 |
| 收藏 | 收藏、取消收藏、查看我的收藏 | 已登录普通用户 |
| 商品维护 | 新增、编辑、软删除商品 | 管理员 |
| 商品状态管理 | 商品上架、下架、状态筛选 | 管理员 |
| 管理员登录 | 后台身份认证和权限进入 | 管理员 |

## 3. 角色权限边界

| 角色 | 是否需要登录 | 可读公开商品 | 可收藏 | 可写商品数据 | 可上下架 | 可访问后台 |
| --- | --- | --- | --- | --- | --- | --- |
| 游客 | 否 | 是 | 否 | 否 | 否 | 否 |
| 已登录普通用户 | 是 | 是 | 是 | 否 | 否 | 否 |
| 管理员 | 是 | 可通过后台查看 | 否 | 是 | 是 | 是 |

## 5. 页面访问范围说明

### 5.1 游客

* 可访问：公开商品页、小程序首页、小程序详情页
* 不可访问：我的收藏页、所有后台页面

### 5.2 已登录普通用户

* 可访问：游客全部页面、我的收藏页
* 不可访问：所有后台管理页面

### 5.3 管理员

* 可访问：后台登录页、后台首页、分类管理、商品管理、SKU 管理、图片管理
* 可选访问：PC Web 新增/编辑表单页，用于课程联调展示
* 不使用：普通用户收藏流程

## 6. 最小可验收方案

为降低需求歧义，本轮采用以下最小可验收方案：

* 商品分类按单级分类设计
* 普通用户只覆盖“收藏/取消收藏/查看收藏”三项登录后行为
* 管理端只覆盖商品、分类、SKU、图片和上下架，不扩展订单与支付
* PC Web 的新增/编辑页面视为课程表单页，不等同于完整商家系统

## 7. 待确认事项

* PC Web 是否需要真正开放给普通用户做登录收藏，还是仅演示公开浏览和表单页
