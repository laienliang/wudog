## 4. 已完成的真实联调验证

### 4.1 后端与 Swagger

- 已验证 `http://127.0.0.1:3000/api/swagger/json` 可返回真实 OpenAPI 文档。
- 文档中存在 `wudong` 标签，说明课程业务模块已被 Swagger 收录。

### 4.2 用户端接口

- `GET /api/app/wudong/product-categories/list` 可返回启用分类。
- `GET /api/app/wudong/products/page?page=1&pageSize=10` 可返回可售商品分页。
- `GET /api/app/wudong/products/detail?id=3` 已确认包含：
  - `craftIntro`
  - `inheritorName`
  - `inheritorIntro`
  - `images`
  - `skus`
  - `soldOut`
  - `isFavorite`

### 4.3 收藏链路

- 普通用户可通过 `POST /api/app/user/login/password` 真实登录。
- 登录后调用 `POST /api/app/wudong/favorites/add` 收藏成功。
- 调用 `GET /api/app/wudong/favorites/page?page=1&pageSize=10` 可返回收藏商品。

### 4.4 管理端影响用户端可见性

- 已通过真实管理接口调用：
  - `POST /api/admin/wudong/products/status`
- 实测结果：
  - 商品 `3` 下架后，用户端商品分页 `total` 从 `1` 变为 `0`
  - 商品 `3` 重新上架后，用户端商品分页 `total` 恢复为 `1`
- 该结果说明后台商品状态变化已能立即影响用户端展示。

### 4.5 PC Web 页面

- 已在浏览器中验证：
  - 分类与商品列表可见
  - 详情页可见工艺介绍与传承人信息
  - 可执行收藏
  - “我的收藏”页可见已收藏商品

## 7. 建议验收顺序

1. 启动 `backend`
2. 打开 Swagger 确认接口文档存在 `wudong`
3. 打开 `pc-web`，用普通用户账号登录
4. 演示“分类 -> 列表 -> 详情 -> 收藏 -> 我的收藏”
5. 登录 `admin-web`，对商品做下架或上架操作
6. 刷新 `pc-web`，确认商品可见性同步变化
7. 在微信开发者工具内复现列表、详情、收藏流程

## 8. 已知限制

- 小程序默认请求 `http://127.0.0.1:3000/api`，这适合微信开发者工具本地调试，不适合真机直接访问电脑本机。
- `admin-web` 的最终人工验收依赖验证码登录，不应把未实际登录的页面检查写成“已完成”。
- Docker 部署仍属于后置阶段，不计入本轮四端联调完成条件。
