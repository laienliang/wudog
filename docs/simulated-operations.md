## 一、AI 模拟数据生成

### 1.1 数据生成方式

在项目开发阶段，利用 AI 辅助生成覆盖全部 6 大模块的模拟数据，数据直接插入 MySQL 数据库，三端（PC端、管理后台、小程序）共享同一份数据源，确保业务流程演示时数据的一致性。

| 模块 | 数据项 | 数量 | 详细说明 |

## 二、PC 端业务流程演练

### 2.1 功能一：首页浏览

#### 2.1.1 Hero Banner 展示

打开 PC 端 `http://localhost:3000`，进入首页。

页面顶部展示全屏宽度的 Hero Banner 区域：
- **背景**：乌东苗寨封面图，覆盖深色渐变遮罩确保文字清晰可见
- **标题**："探寻苗寨之美"（60px 衬线字体，字间距8px，带文字阴影）
- **副标题**："苗族非遗文化 · 千户苗寨 · 梯田风光 · 长桌盛宴"
- **快捷入口**：四个毛玻璃效果按钮（探苗寨/品苗宴/住苗居/购苗礼），hover 时上浮3px

[截图：PC端首页Hero Banner]

#### 2.1.2 服务入口模块

Banner 下方展示五个纯白卡片风格的服务入口，采用 5 列 CSS Grid 等宽排列：
- 民族服饰（🧣）：苗绣·蜡染·银饰 → 点击跳转 `/clothing`
- 餐饮美食（🍜）：苗家特色·长桌宴 → 点击跳转 `/food`
- 民宿预订（🏡）：吊脚楼·梯田景观 → 点击跳转 `/accommodation`
- 苗寨出游（🗺️）：景区门票·路线套餐 → 点击跳转 `/travel`
- 社区分享（📸）：游记·照片·互动 → 点击跳转 `/community`

卡片 hover 时上浮6px，阴影加深。整体宽度与下方内容严格对齐（max-width:1200px，居中）。

[截图：服务入口五个卡片]

### 2.2 功能二：衣·非遗商品

#### 2.2.1 商品列表

点击"购苗礼"或导航栏"民族服饰"，进入 `/clothing` 页面。

| 功能 | 操作方式 | 页面反馈 |
|:-----|:---------|:---------|
| 分类筛选 | 顶部导航栏点击分类标签 | 按分类过滤商品列表 |
| 排序切换 | 点击排序下拉框 | 按销量/价格/评分重新排列 |
| 搜索 | 搜索框输入关键词 | 按名称模糊筛选 |
| 分页 | 底部分页组件 | 切换页码 |

[截图：商品列表页]

#### 2.2.2 商品详情

点击商品卡片进入 `/clothing/detail/:id`。

| 功能 | 说明 |
|:-----|:------|
| 商品主图 | 左侧大图 + 右侧缩略图 |
| 商品信息 | 名称、价格(¥128.00橙色加粗)、评分(4.9★)、销量(23) |
| 工艺介绍 | 苗族平绣技法描述 |
| SKU选择 | 大号/中号/小号，选择后更新价格 |
| 数量选择 | InputNumber，最小1，最大=库存 |
| 加入购物车 | 调用购物车API，右上角显示角标 |
| 收藏 | 点击心形图标，保存到localStorage |
| 评价列表 | Tab切换全部/好评/中评/差评 |

[截图：商品详情页]

#### 2.2.3 购物车

点击右上角购物车图标进入 `/cart`。

| 功能 | 说明 |
|:-----|:------|
| 商品列表 | 展示加购商品，含缩略图/名称/SKU/单价/数量/小计 |
| 数量调整 | InputNumber 增减，实时更新小计 |
| 删除 | 点击删除图标，Popconfirm 确认 |
| 合计金额 | 实时计算选中商品总价 |
| 去结算 | 跳转订单确认页（需登录） |

[截图：购物车页]

#### 2.2.4 收藏夹

点击"收藏夹"按钮进入 `/favorites`，展示收藏的商品列表，支持取消收藏。

[截图：收藏夹页]

### 3.3 功能三：商家入驻审核

- **已通过**：展示已通过的申请
- **已驳回**：展示已驳回的申请

| 区域 | 内容 |

| 资质材料 | JSON格式展示的商家资质（自动解析格式化） |

## 五、模拟运营暴露的问题

### 5.1 数据完整性问题

| 问题 | 涉及模块 | 影响 | 修复方式 |
|:-----|:---------|:-----|:---------|
| userId=0 | 评论/游记/举报 | 匿名显示 | UPDATE 为 user_id=1 |
| scenic_id=0 | 门票 | 景区显示为空 | UPDATE 关联实际景区ID |
| homestay_id=0 | 房型 | 民宿价格显示"??" | UPDATE 关联实际民宿ID |
| merchant_id=0 | 餐厅/民宿 | 无法关联商家 | 种子数据写入时修复 |

### 5.2 功能联调问题

| 问题 | 发现场景 | 根因 | 修复 |
|:-----|:---------|:-----|:------|
| JSON字段.map()报错 | 餐饮管理页白屏 | simple-json返回字符串非数组 | 增加Array.isArray+JSON.parse |
| 表格列数据为空 | 财务结算页 | 前后端字段名不匹配 | 对齐dataIndex |
| 角色删除按钮无效 | 角色管理 | 后端缺少deleteRole路由 | 新增DEL接口 |
| 下拉筛选失效 | 商家/订单管理 | valueType:select改变render参数 | 改用record取值 |

### 5.3 部署与环境问题

| 问题 | 说明 |
|:-----|:------|
| 图片加载失败 | 商品/民宿图片使用本地URL，需启动图片服务 |
| MySQL编码 | 需配置utf8mb4，否则中文显示乱码 |
| 端口占用 | 3000/3001/7001端口需确保未被占用 |
| Redis未启动 | Token黑名单功能依赖Redis服务 |
