# 07 管理后台页面与运营功能设计

## 1. 技术栈

* React 18
* Vite
* Ant Design
* Axios

约束说明：

* Cool Admin Midway 只作为后端脚手架。
* 管理后台前端继续使用 React + Ant Design，不改为 Vue、cool-admin-vue 或 Element Plus。

## 2. 页面清单

| 页面 | 目标 |
| --- | --- |
| 管理员登录页 | 获取管理员 JWT |
| 管理后台布局 | 提供导航、路由保护、基础信息展示 |
| 商品管理页 | 分页、搜索、筛选、上下架、软删除 |
| 商品新增和编辑页 | 维护商品基础信息、工艺介绍、传承人信息 |
| 分类管理页 | 维护商品分类 |
| SKU 管理页 | 维护 SKU 价格、库存、状态 |
| 商品图片管理页 | 维护图片 URL、类型和排序 |

## 3. 页面设计

## 3.1 管理员登录页

### 页面目标

* 管理员输入账号密码并获取后台访问令牌
* 登录能力优先复用 Cool Admin Midway 现有管理员认证体系

### 页面区域

* 登录表单区
* 错误提示区

### 展示字段

* 用户名
* 密码

### 调用接口

* `POST /api/admin/auth/login`

### 交互操作

* 输入账号密码
* 点击登录

### 校验规则

* 用户名必填
* 密码必填

## 4. 商品上架和下架专项设计

商品上架和下架是第 1 组管理后台额外功能，必须单独设计。

### 4.1 操作入口

* 商品管理表格操作列提供“上架”或“下架”按钮
* 商品编辑页状态区也可提供状态操作入口

### 4.2 二次确认

* 上架前弹出确认框，提醒将对用户端可见
* 下架前弹出确认框，提醒用户端将不再正常展示

### 4.3 状态字段

* 使用 `product.status`
* `1` 表示已上架
* `0` 表示已下架

### 4.4 后端接口

* 上架：`PATCH /api/admin/products/{productId}/publish`
* 下架：`PATCH /api/admin/products/{productId}/unpublish`

### 4.5 操作成功后的刷新

* 刷新当前商品列表
* 同步更新行内状态标签

### 4.6 操作失败提示

* 上架校验失败时展示具体原因
* 网络失败时提示稍后重试

### 4.7 上架前校验

至少校验：

* 商品存在且未删除
* 分类有效
* 主图存在
* 工艺介绍存在
* 传承人信息存在
* 至少存在一个启用且未删除的 SKU

### 4.8 下架后的用户端表现

* PC Web 商品列表和详情不再正常展示该商品
* 小程序商品列表和详情不再正常展示该商品
* 已收藏记录可保留，但若访问详情，应收到不可见或下架反馈

## 5. 登录态与前端基础设施

## 5.1 token 存储方式

* 管理端 token 存储在 `localStorage`
* 页面初始化时从本地读取
* 退出登录时清空

## 5.2 Axios 请求拦截器

* 自动注入 `Authorization: Bearer <token>`
* 统一附加 JSON 请求头

## 5.3 Axios 响应拦截器

* 统一处理 `code != 200`
* 统一弹出错误消息
* 对 `401` 做登录态失效处理

## 5.4 未登录跳转

* 若未检测到 token，直接跳转管理员登录页

## 5.5 401 处理

* 清空本地 token
* 提示“登录已失效，请重新登录”
* 跳转登录页

## 5.6 路由保护

* 登录页为公开路由
* 其他后台页面全部为受保护路由

## 6. 最小可验收范围

* 只做单管理员角色
* 不扩展审计日志、多角色权限、图片上传服务
* 图片管理先按 URL 录入，不接入对象存储
