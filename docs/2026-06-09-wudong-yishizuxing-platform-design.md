## 2. 项目概述

### 2.1 项目背景

乌东村是贵州黔东南苗族侗族自治州特色苗寨，拥有苗族银饰锻造、蜡染刺绣、苗家长桌宴、特色民宿、苗寨梯田与节庆文化等文旅资源。现有运营以线下为主，缺乏统一的线上服务入口。本平台以"衣食住行"为业务主线，覆盖游客从内容种草、社区分享、商品/餐饮/住宿/门票预订到订单管理、平台运营的全链路。

### 2.2 项目目标

1. 为游客提供一站式"衣、食、住、行、社区"线上服务
2. 为商家提供订单、商品/房源/票务/座位管理后台
3. 为平台运营方提供用户管理、数据看板、内容审核能力
4. 沉淀乌东文旅数字化资产，支撑后续 AI 智能体接入与商业化扩展

### 2.3 业务范围

- **衣**：苗族非遗手工艺品（银饰、蜡染、刺绣、苗族服饰等）在线展示与购买
- **食**：苗家特色餐厅展示、餐位预订、农产品/特产购买
- **住**：苗寨特色民宿/客栈搜索与预订
- **行**：景区门票、苗寨游路线套餐（一日游/两日游）购买与电子凭证
- **社区**：游客照片上传、游记发布、点赞评论、关注互动
- **管理后台**：用户、商家、订单、内容、数据、运营的全局管理

### 2.4 三端范围

| 终端 | 主要用户 | 核心定位 |
|------|---------|---------|
| 微信小程序端 | 游客 | 移动端主入口，轻量便捷，扫码即用 |
| PC 网页端 | 游客、商家 | 桌面端浏览与下单，展示更丰富 |
| 管理后台（Web） | 商家、平台管理员 | 后台管理与运营 |

```
┌──────────────────────────────────────────────┐
│   客户端：小程序端   PC 网页端   管理后台     │
└──────────────────────┬───────────────────────┘
                       │ HTTPS / JSON
┌──────────────────────▼───────────────────────┐
│   API 网关层（统一鉴权 / 限流 / 日志）         │
└──────────────────────┬───────────────────────┘
                       │
┌──────────┬──────────┼──────────┬─────────────┐
│ 模块一服务 │ 模块二服务 │ 模块三服务 │ 模块四服务 │ 模块五服务 │ 模块六服务
│ 衣-非遗  │ 食-餐饮  │ 住-住宿  │ 行-订票   │ 社区-相册 │ 平台管理
└────┬─────┴────┬─────┴────┬─────┴────┬──────┴────┬─────┘
     │          │          │          │           │
┌────▼──────────▼──────────▼──────────▼───────────▼─────┐
│        公共服务层：用户/认证/订单/支付/上传/消息        │

└───────────────────────────────────────────────────────┘
```

### 4.2 模块边界

6 个模块边界互不重叠，按业务领域划分：

| 组别 | 模块 | 业务领域 | 数据归属 |
|------|------|---------|---------|
| 第 1 组 | 衣——非遗商品 | 商品浏览、加购、下单、评价 | 商品库、订单子表、评价表 |
| 第 2 组 | 食——餐饮美食 | 餐厅/特产浏览、餐位预订、特产下单 | 餐厅表、餐位表、农产品表、订单子表 |
| 第 3 组 | 住——住宿预订 | 民宿搜索、房态日历、预订 | 房源表、房态表、订单子表 |
| 第 4 组 | 行——线路订票 | 门票购买、路线套餐购买、电子票 | 票种表、线路表、电子票表、订单子表 |
| 第 5 组 | 社区——照片分享 | 游记发布、互动、关注 | 游记表、图片表、评论表、关注表 |
| 第 6 组 | 平台管理后台 | 全局用户/订单/内容/数据管理 | 管理相关表、统计视图 |

### 4.3 公共能力依赖

所有模块共同依赖以下公共服务（由各组共同遵守的接口约定）：

| 公共服务 | 提供方 | 调用方 |
|---------|-------|-------|
| 用户注册/登录 | 公共（各组对接） | 全部模块 |
| 用户信息 | 公共 | 全部模块 |
| 统一订单 | 公共（统一订单中心） | 模块 1-4 |
| 统一购物车 | 公共 | 模块 1、2（特产） |
| 统一支付 | 公共 | 全部下单模块 |
| 图片/视频上传 | 公共（OSS） | 全部模块 |
| 消息通知 | 公共 | 全部模块 |
| 搜索服务 | 公共（ElasticSearch/DB 检索） | 全部模块 |

> **约定**：第 1-5 组在设计 API 时应优先调用公共服务接口，避免重复造轮子；公共服务的接口契约由项目指导老师统一定义后下发。

## 5. 公共功能需求

### 5.1 统一用户体系

#### 5.1.1 注册与登录

- **注册方式**：手机号 + 短信验证码
- **登录方式**：手机号 + 密码 / 手机号 + 验证码 / 微信授权登录
- **密码规则**：8-20 位，至少含字母与数字
- **同一手机号/微信号仅能注册一个账号**

#### 5.1.2 个人中心

| 功能 | 说明 |
|------|------|
| 我的资料 | 头像、昵称、性别、地区、个人简介 |
| 我的订单 | 统一展示衣/食/住/行 4 类订单（按 Tab 切换） |
| 我的收藏 | 商品/餐厅/民宿/线路/游记的收藏聚合 |
| 我的相册 | 我发布过的照片与游记管理 |
| 我的评价 | 我对各类商品/服务的评价管理 |
| 收货地址 | 衣/食（特产）模块配送使用 |
| 消息通知 | 系统消息、订单消息、互动消息 |
| 设置 | 账号安全、隐私、退出登录 |

#### 5.1.3 商家入驻

- 游客可申请成为商家，提交资质材料（身份证、营业执照、店铺信息）
- 平台管理员审核通过后开通商家权限
- 商家按所属模块分组，每组只能管理自己模块的店铺

### 5.2 统一购物车

- 衣（实物商品）、食（农产品特产）共用购物车
- 民宿/餐位/门票/线路为预订类商品，**不进入购物车**，直接生成订单
- 购物车支持商品增删改、规格切换、库存校验、失效提醒

### 5.3 统一订单

#### 5.3.1 订单类型

| 订单类型 | 来源模块 | 关键字段 |
|---------|---------|---------|
| 商品订单 | 模块 1、模块 2（特产） | 商品列表、收货地址、物流 |
| 餐位预订 | 模块 2 | 餐厅、日期、时段、人数 |
| 住宿预订 | 模块 3 | 民宿、入住/离店日期、房型 |
| 门票订单 | 模块 4 | 票种、日期、人数、电子票 |
| 路线订单 | 模块 4 | 路线套餐、出发日期、人数 |

#### 5.3.2 订单状态机

```
待支付 → 已支付/待确认 → 已确认 → 进行中（入住中/出行中）→ 已完成
  ↓          ↓               ↓           ↓
已取消    已退款         已退款       已退款/已评价
```

#### 5.3.3 订单操作

- 支付（微信支付）
- 取消订单（未支付/已支付未确认）
- 申请退款（不同模块退款规则见各模块说明）
- 确认收货（实物商品）
- 申请发票（可选）

### 5.4 统一支付

- 微信支付（小程序端必选，PC 端可增加支付宝）
- 支付状态回调查询
- 退款原路返回
- 商家分账记账（平台抽佣比例可配置）

### 5.5 统一图片/视频上传

- 支持图片（jpg/png/webp，单张 ≤ 5MB）
- 支持视频（mp4，单个 ≤ 100MB，时长 ≤ 60s，用于社区游记）
- 自动生成多尺寸缩略图
- OSS 对象存储
- 图片内容审核（涉黄、涉政、暴恐识别）

### 5.6 统一消息通知

- 站内消息（系统公告、订单状态）
- 微信模板消息（小程序端）
- 短信验证码（注册/支付/退款关键节点）
- 通知统一在"消息中心"展示

### 5.7 统一搜索

- 商品/餐厅/民宿/线路/游记均支持关键词搜索
- 搜索结果支持按热度/价格/评分/时间排序
- 搜索历史记录（用户级）
- 热搜词推荐（管理员后台可配置）

### 5.8 首页与导航

#### 5.8.1 小程序端首页

- 顶部 Banner 轮播（管理员配置）
- 金刚区入口：衣、食、住、行、社区、我的
- 热门推荐（人工推荐 + 算法推荐）
- 公告/活动横幅
- 底部 Tab：首页、分类、消息、我的

#### 5.8.2 PC 端首页

- 顶部导航栏：衣、食、住、行、社区、搜索、登录/个人中心
- Banner 大图区
- 各模块入口卡片
- 热门内容/产品推荐区

### 5.9 内容安全

- 用户上传文本/图片必须经内容安全审核
- 审核失败的内容进入待审核状态，商家/管理员可处理
- 敏感词库由平台管理员维护

## 6. 模块一：衣——非遗商品模块

> **开发组**：第 1 组
> **业务定位**：苗族非遗手工艺品的电商交易

### 6.1 业务概述

展示和销售苗族银饰、蜡染、刺绣、苗族服饰等非遗手工艺品。商品具有强文化属性，详情页需重点展示工艺、文化故事、传承人信息。

### 6.2 数据实体

| 实体 | 关键字段 |
|------|---------|
| 商品分类 | 分类ID、分类名称、图标、排序、状态 |
| 商品 | 商品ID、标题、副标题、分类ID、商家ID、主图、价格、市场价、库存、销量、详情（富文本）、工艺介绍、传承人ID、状态、创建时间 |
| 商品 SKU | SKU ID、商品ID、规格名（如"银饰-手镯-中号"）、价格、库存、图片 |
| 商品图片 | 图片ID、商品ID、图片URL、排序 |
| 评价 | 评价ID、订单ID、商品ID、用户ID、评分、文字、图片、追评、创建时间、商家回复 |
| 收藏 | 用户ID、商品ID、创建时间 |

### 6.3 小程序端功能

| 功能 | 说明 |
|------|------|
| 分类浏览 | 一级分类（银饰/蜡染/刺绣/服饰/其他）+ 二级分类 |
| 商品列表 | 支持按分类、价格区间、销量、评分筛选；分页加载；下拉刷新 |
| 商品搜索 | 关键词搜索 + 搜索历史 + 热门搜索 |
| 商品详情 | 主图轮播 + 标题/价格/库存/销量/评分 + 工艺介绍 + 传承人故事 + 规格选择 + 评价列表 + 详情富文本 |
| 加入购物车 | 选择规格后加入购物车 |
| 立即购买 | 选择规格后直接下单 |
| 收藏 | 收藏/取消收藏 |
| 评价 | 已购用户可对商品评价（文字+图片+评分），可追评 |
| 联系客服 | 跳转平台客服 |

### 6.4 PC 端功能

| 功能 | 说明 |
|------|------|
| 分类筛选页 | 多条件组合筛选（分类、价格、评分、风格、材质） |
| 商品详情大图页 | 大图展示、富文本详情、规格选择、加入购物车 |
| 购物车 | 增删改、失效商品提示、合计金额、去结算 |
| 收藏夹 | 我的商品收藏列表 |
| 评价中心 | 我的评价与商家回复 |

### 6.5 管理后台功能

| 功能 | 说明 |
|------|------|
| 分类管理 | 增删改查、排序、上下架 |
| 商品管理 | 新增/编辑/上下架/批量导入；商品图片、规格、详情管理 |
| 库存管理 | 各 SKU 库存调整、库存预警（< 10 提醒） |
| 订单管理 | 订单列表、详情查看、发货、修改物流、退款审核 |
| 评价管理 | 评价列表、回复、隐藏、删除 |
| 数据统计 | 销售额、订单量、热销商品 TOP10、评价数 |

### 6.6 业务规则

- 实物商品需填写运费模板（按重量/件数）
- 7 天无理由退换（特殊定制商品除外）
- 评价完成后 30 天内可追评一次
- 库存为 0 时前端不可下单

## 7. 模块二：食——餐饮美食模块

> **开发组**：第 2 组
> **业务定位**：苗家特色餐厅展示与预订 + 农产品特产电商

### 7.1 业务概述

包含两大子业务：
1. **餐饮预订**：苗家特色餐厅的展示与餐位预订
2. **农产品特产**：茶叶、腊肉、米酒、酸食等农产品的在线购买

### 7.2 数据实体

| 实体 | 关键字段 |
|------|---------|
| 餐厅 | 餐厅ID、名称、所属商家、地址、坐标、营业时间、容纳人数、餐厅介绍、主图、状态 |
| 餐厅菜品 | 菜品ID、餐厅ID、菜品名、价格、主图、介绍、是否招牌 |
| 餐位时段 | 时段ID、餐厅ID、时段名（如"午餐 11:30-13:30"）、最大预订数 |
| 餐位预订订单 | 订单ID、餐厅ID、用户ID、预订日期、时段、人数、姓名、电话、备注、状态 |
| 农产品分类 | 分类ID、名称、图标 |
| 农产品商品 | 商品ID、分类ID、名称、价格、规格、库存、主图、产地、保质期、详情 |
| 农产品订单 | 订单ID（同统一订单） |
| 评价 | 评价ID、餐厅/商品ID、用户ID、评分、文字、图片 |

### 7.3 小程序端功能

| 功能 | 说明 |
|------|------|

> 商家后台的具体业务管理页面（商品管理、订单管理、房源管理等）由对应业务模块组（第 1-4 组）开发，**作为商家后台的子模块嵌入**。模块 5 的商家不直接管理商品，所以商家后台的商品管理页由模块 1 组开发并嵌入。

### 11.5 业务规则

- 商家入驻审核 SLA：3 个工作日内
- 财务结算周期：T+7（每 7 天结算一次）
- 平台抽佣比例默认：实物商品 5%，服务类（民宿/门票/餐位/路线）10%
- 操作日志保留 1 年
- 敏感词命中后内容进入待审核状态，命中 3 次以上用户自动禁言 24 小时

## 12. 非功能需求

### 12.1 性能需求

| 指标 | 要求 |
|------|------|
| 首页加载时间 | ≤ 2 秒（4G 网络） |
| API 响应时间 | 95% 请求 ≤ 500ms |
| 并发用户数 | 支持 1000 并发用户 |
| 图片加载 | 缩略图 ≤ 200KB，原图懒加载 |
| 搜索响应 | ≤ 1 秒返回结果 |

### 12.2 安全需求

- 密码使用 bcrypt 加密存储
- 敏感信息（身份证、手机号）脱敏展示
- API 接口使用 HTTPS
- 防 SQL 注入（使用 ORM 参数化查询）
- 防 XSS 攻击（富文本过滤）
- 防 CSRF 攻击
- 操作日志全记录
- 关键操作（支付、退款）二次验证

### 12.3 可用性

- 系统可用性 ≥ 99.5%
- 关键服务故障恢复时间 ≤ 30 分钟
- 数据每日凌晨自动备份，保留 30 天

### 12.4 兼容性

| 端 | 兼容范围 |
|----|---------|
| 小程序 | 微信 8.0+ |
| PC 端 | Chrome 90+、Edge 90+、Safari 14+ |
| 管理后台 | Chrome 90+、Edge 90+ |
| 屏幕适配 | 移动端：375px - 414px；PC 端：1280px+ |

### 12.5 国际化

- 当前版本仅支持简体中文
- 预留多语言扩展接口

### 12.6 数据统计与分析

- 用户行为埋点（PV、UV、点击、转化）
- 关键漏斗分析（浏览→加购→下单→支付）
- 数据看板可视化（Echarts / 图表库）
- 每周/每月自动生成运营报表
