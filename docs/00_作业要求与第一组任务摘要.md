# 00 作业要求与第一组任务摘要

## 1. 项目背景

* 项目名称：乌东文旅平台 AI 全栈开发实战
* 小组身份：第 1 组“衣·非遗商品模块”
* 业务范围：银饰、蜡染、刺绣等非遗商品展示与电商相关基础功能
* 当前阶段：仅进行文档设计，不编写业务代码

## 2. 四端交付物

| 端 | 目标产物 | 说明 |
| --- | --- | --- |
| backend | Midway.js + Cool Admin Midway 服务 | 基于 Cool Admin Midway 脚手架提供公开接口、收藏接口与后台管理接口 |
| web | React 18 + Vite PC Web | 展示商品列表、详情，以及课程要求的新增/编辑表单页 |
| miniprogram | 原生微信小程序 | 面向普通用户展示商品并支持收藏 |
| admin | React 18 + Vite + Ant Design 后台 | 面向管理员进行分类、商品、SKU、图片与上下架管理 |
| docs | 需求、设计、接口、测试与验收文档 | 当前轮次唯一可交付内容 |

## 3. 技术栈

* 后端：Midway.js、Cool Admin Midway 脚手架
* 数据库：MySQL
* 缓存：Redis
* 接口文档：Swagger / OpenAPI
* 容器化：Docker
* 协作：Git + GitHub
* PC Web：React 18、Vite、React Router v6、Axios
* 微信小程序：原生小程序、`wx.request`
* 管理后台：React 18、Vite、Ant Design、Axios

## 4. 端口

| 模块 | 端口 | 备注 |
| --- | --- | --- |
| 后端 | `3000` | 所有客户端共用 |
| PC Web | `5173` | 本地开发端口 |
| 管理后台 | `5174` | 本地开发端口 |
| 微信小程序 | 无固定端口 | 在微信开发者工具中本地运行，调用后端 `3000` |

## 5. 公共接口规范

* 接口前缀统一为 `/api`
* 路径命名统一使用全小写和连字符
* 公开商品列表、商品详情接口不强制登录
* 收藏接口、写操作接口、后台管理接口统一使用 JWT Bearer Token
* 统一响应结构：

```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

* 分页数据结构统一包含 `total`、`page`、`pageSize`、`list`

## 10. 第 1 组特殊业务规则

* 商品库存只存放在 `product_sku`，不存放在 `product`
* 一个商品可以有多个 SKU
* 每个 SKU 有独立价格、规格、库存和启用状态
* 当某个 SKU 库存为 `0` 时，接口必须返回售罄状态
* 商品详情必须包含工艺介绍 `craft_intro`
* 商品详情必须包含传承人相关信息 `inheritor_name`、`inheritor_intro`
* 收藏关系必须做去重校验
* 删除默认使用软删除
* 默认查询必须过滤软删除数据
* 管理后台支持商品上架、下架
* 下架商品不得在普通用户端正常展示
* 普通商品列表、详情接口不强制登录
* 收藏接口、写接口和后台接口必须校验 JWT Bearer Token
* 后端开发必须基于 Cool Admin Midway，优先复用其管理员登录、JWT、角色权限、菜单、统一响应和基础 CRUD
