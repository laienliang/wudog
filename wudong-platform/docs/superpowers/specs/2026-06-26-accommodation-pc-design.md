# 住宿模块 PC 端设计文档

> 对应需求：需求规格说明书第8章「住——住宿预订模块」PC 端功能
> 页面：民宿列表 / 民宿详情 / 预订页

---

## 路由设计

| 路径 | 页面 | 说明 |
|------|------|------|
| `/accommodation` | 民宿列表页 | 搜索+筛选+卡片网格 |
| `/accommodation/:id` | 民宿详情页 | 封面/介绍/房型列表/设施 |
| `/accommodation/booking/:id` | 预订页 | 选房型+日期+信息+提交 |

## 页面设计

### 1. 民宿列表页 (`index.tsx`)
- 顶部搜索框（按名称搜索）
- 排序：评分、价格
- 民宿卡片网格（4列）：封面图 + 名称 + 评分(⭐) + 设施标签(Tag) + 价格区间
- 点击卡片跳转到详情页
- 加载态骨架屏、空态提示

### 2. 民宿详情页 (`detail.tsx`)
- 封面大图（Image 组件，可预览）
- 民宿信息区：名称 + 评分 + 电话 + 地址
- 设施标签列表
- 民宿介绍文字
- 房型列表表格：房型名 / 床型 / 面积 / 可住人数 / 价格 / 「预订」按钮
- 点击「预订」跳转到预订页

### 3. 预订页 (`booking.tsx`)
- 民宿名 + 房型选择（默认选中）
- 入住日期 + 离店日期（Input date）
- 联系人姓名 + 手机号
- 备注
- 提交按钮 → 调用 POST `/api/v1/orders`（复用统一订单）
- 提交成功后显示订单号

## 技术方案
- React + Ant Design + axios
- API 通过 `/api/v1` proxy 到后端 7001
- 图片通过 `/images/...` 从 web-pc 静态服务加载
- 统一订单：`order_type = 'accommodation'`

## 文件结构

```
packages/web-pc/src/pages/accommodation/
├── index.tsx       # 列表页
├── detail.tsx      # 详情页
└── booking.tsx     # 预订页
```
