# 小程序联调说明

## 1. 适用范围

- 本说明用于微信开发者工具内联调 `miniapp/`
- 当前默认后端地址为 `http://127.0.0.1:3000/api`

## 2. 打开方式

1. 启动本机后端：`backend -> npm run start:local`
2. 打开微信开发者工具
3. 选择“导入项目”
4. 项目目录选择：`D:\临时文件\水课任务\人工智能实践\miniapp`
5. 使用测试 `AppID` 或当前 `project.config.json` 中的 `touristappid`

## 3. 当前配置说明

- `project.config.json` 已设置 `"urlCheck": false`
- `app.js` 中的 `baseUrl` 应保持指向本机后端
- 当前链路默认用于开发者工具本地调试

## 4. 演示账号

- 普通用户：`13800000000 / 123456`

## 5. 人工联调检查清单

### 5.1 首页和分类

- 首页能正常打开
- 分类列表可见 `前端联调分类`
- 点击分类后能刷新商品列表

### 5.2 商品列表和详情

- 商品列表可见 `乌东前端联调商品`
- 进入详情页后可看到：
  - 工艺介绍
  - 传承人姓名
  - 传承人介绍
  - 商品图片
  - SKU 信息
  - 售罄/可售状态

### 5.3 登录与收藏

- 用 `13800000000 / 123456` 登录成功
- 在详情页点击收藏成功
- 收藏页能看到已收藏商品
- 取消收藏后，收藏页同步消失

### 5.4 与管理端联动

- 当后台把商品下架后，小程序商品列表和详情应不可见
- 当后台重新上架后，小程序商品列表应恢复可见

## 6. 已知限制

- 真机不能直接访问电脑本机 `127.0.0.1`
- 如果要做真机演示，需要把后端改成局域网可访问地址，并同步修改小程序请求基地址
- 当前阶段的验收目标是“开发者工具内联调通过”，不是“真机联网发布”
